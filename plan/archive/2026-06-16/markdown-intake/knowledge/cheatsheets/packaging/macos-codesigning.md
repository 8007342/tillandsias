# macOS Code Signing & Notarization

## Gatekeeper & Quarantine

Gatekeeper enforces code signing and notarization on downloaded software. Files downloaded via browsers, curl, AirDrop, etc. receive the `com.apple.quarantine` extended attribute. On first launch, Gatekeeper checks this xattr, verifies the signature, and queries Apple's notarization database.

```bash
# Inspect quarantine xattr
xattr -p com.apple.quarantine MyApp.app

# Remove quarantine (local/unsigned builds only)
xattr -cr MyApp.app
xattr -cr MyApp.dmg

# List all xattrs
xattr -l MyApp.app
```

Files built locally never receive the quarantine xattr -- bypass is only needed for downloads or transferred builds.

## Signing Identities

| Identity | Use Case |
|---|---|
| `Developer ID Application: Name (TEAMID)` | Distribution outside App Store |
| `Apple Development: email (ID)` | Local development/testing |
| `-` (ad-hoc) | No identity; local-only, fails Gatekeeper |

```bash
# List available identities
security find-identity -v -p codesigning

# Ad-hoc sign (no Apple account needed, no notarization possible)
codesign --force --sign - MyApp.app
```

## Code Signing with codesign

```bash
# Sign with hardened runtime (required for notarization)
codesign --force --sign "Developer ID Application: Name (TEAMID)" \
  --options runtime \
  --timestamp \
  --entitlements entitlements.plist \
  MyApp.app

# Deep sign (frameworks, helpers, dylibs inside bundle)
codesign --force --deep --sign "Developer ID Application: Name (TEAMID)" \
  --options runtime --timestamp \
  MyApp.app

# Verify signature
codesign --verify --deep --strict --verbose=2 MyApp.app

# Display signing info and entitlements
codesign -dv --verbose=4 MyApp.app
codesign -d --entitlements - MyApp.app
```

**Critical flags:**
- `--options runtime` -- enables Hardened Runtime (mandatory for notarization since macOS 10.15)
- `--timestamp` -- embeds a secure timestamp from Apple's server
- `--force` -- re-sign if already signed

## Entitlements & Hardened Runtime

Hardened Runtime restricts code injection, DYLD hijacking, and memory tampering. Entitlements grant exceptions. Create an `entitlements.plist`:

```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN"
  "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
  <!-- Common entitlements for non-sandboxed apps -->
  <key>com.apple.security.cs.allow-unsigned-executable-memory</key>
  <true/>
  <key>com.apple.security.cs.allow-jit</key>
  <true/>
</dict>
</plist>
```

Common entitlements: `cs.allow-unsigned-executable-memory` (JIT engines, WebKit), `cs.allow-jit`, `cs.disable-library-validation` (loading third-party dylibs), `cs.allow-dyld-environment-variables`.

## Notarization with notarytool

`altool` was retired November 2023. Use `notarytool` exclusively.

```bash
# Submit for notarization (App Store Connect API key -- preferred for CI)
xcrun notarytool submit MyApp.dmg \
  --key /path/to/AuthKey_XXXXXX.p8 \
  --key-id XXXXXX \
  --issuer "issuer-uuid" \
  --wait

# Submit with Apple ID (interactive/local)
xcrun notarytool submit MyApp.dmg \
  --apple-id "dev@example.com" \
  --team-id TEAMID \
  --password "@keychain:AC_PASSWORD" \
  --wait

# Check submission status
xcrun notarytool info <submission-id> --key ... --key-id ... --issuer ...

# Fetch detailed log on failure
xcrun notarytool log <submission-id> --key ... --key-id ... --issuer ...
```

## Stapling

Stapling attaches the notarization ticket to the artifact so Gatekeeper can verify offline.

```bash
xcrun stapler staple MyApp.app
xcrun stapler staple MyApp.dmg

# Validate staple
xcrun stapler validate MyApp.app
spctl --assess --type execute -vvv MyApp.app
```

Staple `.app` before packaging into `.dmg`, or staple the `.dmg` directly.

## .app Bundle Structure

```
MyApp.app/
  Contents/
    Info.plist          # Bundle metadata (required)
    MacOS/
      myapp             # Main executable
    Frameworks/         # Embedded dylibs, frameworks
    Resources/          # Icons, assets
      myapp.icns
    _CodeSignature/     # Generated by codesign
```

**Info.plist requirements:** `CFBundleIdentifier` (reverse-DNS), `CFBundleName`, `CFBundleVersion`, `CFBundleShortVersionString`, `CFBundleExecutable`, `LSMinimumSystemVersion`.

## .dmg Packaging

```bash
# Create DMG from app bundle
hdiutil create -volname "MyApp" -srcfolder MyApp.app \
  -ov -format UDZO MyApp.dmg

# Sign the DMG itself
codesign --force --sign "Developer ID Application: Name (TEAMID)" \
  --timestamp MyApp.dmg
```

## Tauri Bundle Signing

Tauri v2 handles signing and notarization via environment variables:

```bash
# Certificate (local keychain or CI export)
export APPLE_SIGNING_IDENTITY="Developer ID Application: Name (TEAMID)"

# For CI -- base64-encoded .p12 certificate
export APPLE_CERTIFICATE="base64-encoded-p12"
export APPLE_CERTIFICATE_PASSWORD="p12-password"

# Notarization via App Store Connect API (preferred)
export APPLE_API_ISSUER="issuer-uuid"
export APPLE_API_KEY="key-id"
export APPLE_API_KEY_PATH="/path/to/AuthKey.p8"

# Or via Apple ID
export APPLE_ID="dev@example.com"
export APPLE_PASSWORD="app-specific-password"
export APPLE_TEAM_ID="TEAMID"
```

Tauri automatically creates a temporary keychain in CI, imports the certificate, signs with hardened runtime, submits for notarization, and staples the result.

## CI Signing with Keychain

When not using Tauri's built-in automation:

```bash
# Create temporary keychain
security create-keychain -p "" build.keychain
security default-keychain -s build.keychain
security unlock-keychain -p "" build.keychain

# Import certificate
echo "$APPLE_CERTIFICATE" | base64 --decode > cert.p12
security import cert.p12 -k build.keychain -P "$APPLE_CERTIFICATE_PASSWORD" \
  -T /usr/bin/codesign
security set-key-partition-list -S apple-tool:,apple: -s -k "" build.keychain

# Sign, notarize, staple (as above)
# ...

# Cleanup
security delete-keychain build.keychain
rm -f cert.p12
```

## Quick Reference

| Task | Command |
|---|---|
| List identities | `security find-identity -v -p codesigning` |
| Ad-hoc sign | `codesign -fs - App.app` |

| Verify | `codesign --verify --deep --strict App.app` |
| Notarize | `xcrun notarytool submit App.dmg --key ... --wait` |
| Staple | `xcrun stapler staple App.app` |

| Strip quarantine | `xattr -cr App.app` |
