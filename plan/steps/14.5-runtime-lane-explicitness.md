# Step 14.5: Runtime lane explicitness

## Objective

Make the launcher’s runtime ownership model explicit: desktop user-session,
headless service-account, and dev/test wrapper. Interactive launch paths must
preflight the desktop lane before the first Podman call, while the supervised
headless lane must remain tied to the packaged `tillandsias` service account.

## Owned files or file scopes

- `crates/tillandsias-podman/src/lib.rs`
- `crates/tillandsias-headless/src/main.rs`
- `methodology/specs/user-runtime-lifecycle/architecture.md`
- `methodology/specs/user-runtime-lifecycle/TRACES.md`
- `plan/issues/runtime-lane-explicitness.md`

## Dependency tail

- Depends on the tray/router and user-runtime port work staying coherent.
- Must not reintroduce direct Podman construction outside the shared layer.

## Current evidence

- Desktop and service-account lane requirements already exist in OpenSpec.
- The launcher previously inferred ownership implicitly from whatever Podman
  inherited, which made `/run/user/<uid>` ownership failures hard to classify.
- The service-account lane already exists in packaging; the missing piece was a
  shared runtime-lane helper and launcher preflight.
- Verification completed: `cargo test -p tillandsias-podman --lib`,
  `cargo test -p tillandsias-headless --bin tillandsias --no-run`, and
  `cargo test -p tillandsias-headless --bin tillandsias` all passed.

## Next action

- Verify the lane helpers compile and the interactive launchers fail fast when
  the desktop session is missing.
- Keep dev/test wrappers confined to scripts and litmus harnesses.
- Record any new runtime edge case as a methodology event before widening the
  spec.
