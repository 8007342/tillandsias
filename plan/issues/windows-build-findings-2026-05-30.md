**Change made**: bake the short git commit SHA into the diagnose surface as a
new `build_commit: &'static str` field. Operators pasting `--diagnose --json`
into bug reports now have ground-truth on which commit produced the running
binary. The workspace `version` rolls only on release (so two binaries from
the same release tag can differ); `build_commit` closes that gap. Backward-
compatible — additive to a JSON contract that pins key *presence*, not
key-set equality.

**Findings** (free-form):
- The diagnose surface is now richer for triage: `version` answers "what
  release tag is this?" and `build_commit` answers "what specific commit
  produced this binary?". Especially useful between releases when multiple
  dev builds carry the same workspace VERSION but differ at the commit
  level.
- The git-not-available fallback to `"unknown"` is the principled choice:
  a binary built from a release tarball (no `.git/`) still produces a
  valid `--diagnose --json` rather than failing the build.
- Schema-pin litmus test is now self-documenting: the pinned-keys list in
  `diagnose_json_top_level_keys_pinned` doubles as the contract surface.
  Any future refactor that drops `build_commit` surfaces here pre-build,
  same as the existing 10 keys.
