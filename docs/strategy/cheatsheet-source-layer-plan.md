# Cheatsheet source layer — verbatim provenance backing for `cheatsheets/`

@trace spec:cheatsheet-source-layer (NEW capability — to be created), spec:agent-cheatsheets (delta)

**Status**: Design (Opus, 2026-04-26). Implementation pending. Five-chunk plan at the bottom.

## 1. Why this layer exists

The cheatsheet `## Provenance` section ALREADY exists. The failure mode the user is calling out: a cheatsheet author writes a plausible URL into Provenance WITHOUT actually fetching it. The URL might 404, redirect, mean something subtly different, or simply not contain the patterns the cheatsheet "blesses". `cheatsheets/runtime/local-inference.md` (initial commit) was the canonical cautionary tale — its URLs were plausible but unverified. Commit 2026-04-27 fixed it post-fact: `/api/embeddings` was actually superseded by `/api/embed`, and the `OLLAMA_NUM_PARALLEL` claim was undocumented. The cheatsheet was wrong because it was never fetched.

The fix: every cited URL has a verbatim local copy on host disk, content-hashed, license-checked, and indexed. The cheatsheet's `## Provenance` section then carries the local-source path AS WELL AS the URL. CI verifies both sides exist.

The slogan "traceability outwards from Tillandsias" means: a maintainer (or an agent inside the forge with an opt-in mount) can read the SAME bytes the cheatsheet author saw — without round-tripping the network.

## 2. Directory layout

**Location**: `cheatsheet-sources/` at repo root, sibling of `cheatsheets/`.

- Sibling-of-`cheatsheets/` makes the relationship grep-obvious.
- NOT under `docs/` because `docs/` is human-curated narrative; this is a fetched cache.
- NOT under `images/` because it is intentionally NOT shipped into the forge image.
- The hot/cold split (per `docs/strategy/forge-hot-cold-split-plan.md`) is unambiguous: this directory is COLD by definition. Disk only, never tmpfs, never baked into the image.

**Git status**: tracked-with-LFS for the verbatim payloads, plus a fully tracked `INDEX.json` and `.meta.yaml` sidecars.

- Tracking the bytes guarantees offline verification on a fresh clone.
- LFS keeps git pack sizes sane (HTML can be 1–5 MB per page; 60+ sources compresses poorly).
- Sidecar metadata is plain YAML so `git diff` shows license/hash/date changes legibly.
- For sources whose license forbids redistribution (vendor docs not on the allowlist), the verbatim file is `.gitignore`-d but the sidecar `.meta.yaml` is tracked. Anyone running the fetcher locally re-creates the missing file.

`.gitignore` additions:
```
cheatsheet-sources/**/*.html.norepublish
cheatsheet-sources/**/*.pdf.norepublish
```

The `.norepublish` suffix is the "license check failed bundling check" marker.

**Path convention** — deterministic from URL, mirror host + path:
```
cheatsheet-sources/<host>/<path-with-slashes>/<basename>.<ext>
```

Examples:
- `https://owasp.org/www-project-cheat-sheets/cheatsheets/Session_Management_Cheat_Sheet.html` → `cheatsheet-sources/owasp.org/www-project-cheat-sheets/cheatsheets/Session_Management_Cheat_Sheet.html`
- `https://www.rfc-editor.org/rfc/rfc6265.html` → `cheatsheet-sources/www.rfc-editor.org/rfc/rfc6265.html`
- `https://github.com/ollama/ollama/blob/main/docs/api.md` → fetcher REWRITES to raw form: `cheatsheet-sources/raw.githubusercontent.com/ollama/ollama/main/docs/api.md`

**Stored format precedence**:
1. Single-page HTML (preferred per user). Try `?print=1`, `/print/`, `/single-page/` variants.
2. RFC text form for IETF.
3. Markdown when fetching from `raw.githubusercontent.com`.
4. PDF as last resort, paired with `pdftotext` extraction in `<basename>.pdf.txt`.

JS-rendered SPAs flagged with `render: js-required`; resolution path: try print variants, GitHub-source mirror, last resort host-chromium escape hatch.

## 3. Per-source manifest

Per-file `.meta.yaml` sidecar PLUS a central `cheatsheet-sources/INDEX.json`.

**Sidecar format** (`<basename>.<ext>.meta.yaml`):
```yaml
url: https://www.rfc-editor.org/rfc/rfc6265.html
canonical_url: https://www.rfc-editor.org/rfc/rfc6265
fetched: 2026-04-26T14:22:11Z
fetcher_version: 1
content_sha256: 7c9b3f...
content_length: 234112
content_type: text/html; charset=utf-8
http_status: 200
final_redirect: https://www.rfc-editor.org/rfc/rfc6265.html
publisher: IETF / RFC Editor
license: ietf-trust-legal-provisions
license_url: https://trustee.ietf.org/license-info/
redistribution: bundled        # bundled | attribute-only | do-not-bundle
allowlist_match: rfc-editor.org
render: static                  # static | js-required | pdf-only
title: "RFC 6265: HTTP State Management Mechanism"
cited_by:
  - cheatsheets/web/cookie-auth-best-practices.md
notes: ""
```

**Central INDEX.json** is the union of all sidecars in a deterministic shape, regenerated by a script. CI `--check` mode rejects an out-of-date INDEX.

A cheatsheet's `## Provenance` section becomes:
```markdown
## Provenance

- MDN "HTTP cookies" — fetched 2026-04-26, license: CC-BY-SA 2.5,
  local: `cheatsheet-sources/developer.mozilla.org/en-US/docs/Web/HTTP/Cookies.html`
  Source: <https://developer.mozilla.org/en-US/docs/Web/HTTP/Cookies>
- **Last updated:** 2026-04-26
```

Both URL AND local path visible. Maintainers can `cat cheatsheet-sources/...` to re-verify.

## 4. Fetcher tool

`scripts/fetch-cheatsheet-source.sh <URL> [--cite cheatsheets/<path>]`

**Algorithm**:
1. Validate URL: scheme `https://` only, host on the allowlist (`license-allowlist.toml`). Refuse otherwise unless `--manual-review`.
2. Compute deterministic on-disk path from URL.
3. Try single-page variants in order; first that returns 200 + non-empty body wins. RFCs prefer `.txt`. GitHub blob URLs rewritten to `raw.githubusercontent.com`.
4. `curl -L --proto =https --tlsv1.2 -A "tillandsias-cheatsheet-fetcher/1 (+https://github.com/8007342/tillandsias)" --max-time 60 -o <tmp> <url>`. Follow redirects, record final URL.
5. SHA-256 the bytes. Store verbatim. (Tracker-stripping is a separate, opt-in `--canonicalize` pass.)
6. Look up host in `license-allowlist.toml`, derive license/publisher/redistribution.
7. Write `<path>` and `<path>.meta.yaml`. If `redistribution: do-not-bundle`, suffix with `.norepublish`.
8. If `--cite cheatsheets/<path>`, append local-source line to that cheatsheet's Provenance. Idempotent.
9. Update `INDEX.json` (or call `scripts/regenerate-source-index.sh`).

## 5. Cheatsheet ↔ source binding & validation

`scripts/check-cheatsheet-sources.sh`:
1. Parse every cheatsheet's `## Provenance` for URL lines and `local:` paths.
2. For each URL, look up in `INDEX.json`. Missing → ERROR.
3. For each `local:` path, confirm file exists OR `.meta.yaml` exists with `redistribution: do-not-bundle`.
4. Confirm at least one cheatsheet cites every entry in INDEX.json (orphan detection).
5. SHA-check: re-hash present files, compare to manifest. Mismatch → ERROR.

Wired into `openspec validate` chain and pre-commit.

## 6. Hot/cold integration

The cold layer lives on host disk only. NOT bind-mounted into the forge container — the agent's view stops at the cheatsheet's blessed summary (`/opt/cheatsheets/`). The verbatim layer is the operator's accountability proof.

Two paths for an agent that wants to verify:
- **Default**: it cannot. The forge is offline; the cheatsheet IS the contract.
- **Maintainer escalation**: opt-in `forge.mount_source_layer = true` config flag bind-mounts `cheatsheet-sources/` read-only at `/opt/cheatsheet-sources/`. Off by default; for cheatsheet-author workflows. Logged via `accountability` channel.

The cheatsheet `INDEX.md` carries `[verified: <sha256-prefix>]` markers pulled from INDEX.json at index-regen time. Agents see the marker; they don't see the bytes.

## 7. Attribution + license

The Provenance section is the user-visible attribution. The sidecar duplicates for machine consumption. A top-level `cheatsheet-sources/ATTRIBUTION.md` aggregates per-publisher (alphabetical, generated). Downstream redistributors read this to honour licenses.

License rollups (initial allowlist):
- IETF RFCs — IETF Trust Legal Provisions, freely redistributable.
- W3C — W3C Document License (2015), redistributable with attribution.
- WHATWG — CC-BY 4.0.
- MDN — CC-BY-SA 2.5.
- OWASP — CC-BY-SA 4.0.
- PostgreSQL docs — PostgreSQL License (BSD-style).
- Python docs — PSF License.
- Rust docs — MIT or Apache-2.0.
- kernel.org — GPL-2.0+.
- Ollama — MIT.
- AWS / Azure / GCP / Oracle docs — vendor agreements; do-not-bundle (link only).

## 8. Failure modes

- **Source 404 between fetches** — refresh script flags cheatsheet `status: stale`, sets `http_status: 404` in sidecar. Last known good copy stays on disk. INDEX line gets `[STALE]`.
- **License changes** — license check on every fetch. Incompatible license → file renamed `.norepublish`, `redistribution` flips. CI flags every cheatsheet citing it.
- **JS-rendered empty body** — try print variants, GitHub-source mirror, last resort host-chromium escape hatch.
- **Content drift** — SHA mismatch on refresh sets `staleness: drift`; cheatsheet author re-reads both versions, either confirms (bump `Last updated`) or updates content.

## 9. Migration plan

Audit existing cheatsheets:
```bash
scripts/audit-cheatsheet-sources.sh > /tmp/audit.csv
# columns: cheatsheet_path, source_url, in_index_json, license_allowlisted, sha256_present
```

Triage:
- Allowlisted-domain URLs: bulk-fetch automatically.
- Off-allowlist URLs: surface for maintainer review.
- Bad URLs (404 on first fetch): cheatsheet flagged DRAFT; URL replaced/removed.

Effort: ~2 hours of supervised fetcher runs (~1 minute per URL, mostly automatable).
