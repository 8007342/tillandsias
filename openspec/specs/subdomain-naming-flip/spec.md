# subdomain-naming-flip Specification

## Status

status: active
promoted-from: openspec/changes/archive/2026-04-26-subdomain-naming-flip/
annotation-count: 5

## Purpose

Reorganize subdomain URL structure from `<project>.opencode.localhost` to `opencode.<project>.localhost`, enabling hierarchical service discovery (`*.opencode.localhost`, `*.flutter.localhost`, etc.) and simplifying browser-MCP allowlists.

## Requirements

### Requirement: Subdomain ordering inversion

All URLs referencing forge services SHALL use the pattern `<service>.<project>.localhost` instead of `<project>.<service>.localhost`. This groups all services for a single project under a unified namespace.

#### Scenario: Service-first URL structure

- **WHEN** a browser navigates to an OpenCode Web session
- **THEN** the URL SHALL be `opencode.<project>.localhost/` (e.g., `opencode.java.localhost/`)
- **AND** NOT `<project>.opencode.localhost/` (old shape)

#### Scenario: Multiple services per project use consistent hierarchy

- **WHEN** a project has both OpenCode Web and Flutter dev server running
- **THEN** their URLs SHALL be `opencode.project.localhost/` and `flutter.project.localhost/`
- **AND** both fall under the `*.project.localhost` wildcard namespace
- **AND** browser-MCP allowlists can use `*.project.localhost:8080 except opencode.*` syntax

### Requirement: Router configuration update

The Caddyfile generated by `regenerate_router_caddyfile()` SHALL emit rules in the new `<service>.<project>.localhost` format.

#### Scenario: Router rules use new subdomain shape

- **WHEN** a forge is attached to project `acme`
- **THEN** the generated Caddyfile route SHALL be `opencode.acme.localhost:80 { reverse_proxy tillandsias-acme-forge:4096 }`
- **AND** NOT `acme.opencode.localhost:80 { ... }`

### Requirement: Loopback resolution unchanged

The `*.localhost` loopback resolution (RFC 6761) applies at any subdomain depth and requires no configuration change. Browsers and curl automatically resolve any `*.localhost` hostname to `127.0.0.1`.

#### Scenario: RFC 6761 loopback resolution unchanged

- **WHEN** a browser resolves `opencode.java.localhost`
- **THEN** the OS resolver returns `127.0.0.1` automatically (RFC 6761 compliance)
- **AND** no zone file or DNS stub configuration is needed
