## Why

In-forge agents (OpenCode, Claude, OpenSpec) currently land in a fresh container cold: they spend the first prompt scanning `flake.nix`, `Cargo.toml`, `pubspec.yaml`, `.tillandsias/`, and the OpenSpec tree before producing any user-visible work. Every project owns the perfect surface to short-circuit this — `README.md` — but only if it is treated as the agent's bootstrap context, auto-regenerated on every push from authoritative manifests, and structurally enforced. This change makes the README a first-class agent artifact: re-derived from sources of truth rather than human-edited prose, paired with a `/startup` skill that routes the agent into one of three flows (empty project, missing-or-bad README, ready) so prompt 1 always lands on real work.

## What Changes

- **NEW** `/startup` skill — entrypoint command auto-invoked by an OpenCode shim on first prompt of every forge launch. Detects project state (`empty` / `non-empty + bad README` / `non-empty + good README`) and dispatches to the appropriate sub-skill. Output is a single screen.
- **NEW** `/bootstrap-readme-and-project` skill — empty-project welcome flow: prints a project-name banner, lists three curated sample prompts (read from `cheatsheets/welcome/sample-prompts.md`), summarizes forge capabilities in one screen, and ends with "what would you like to do?" without forcing any choice.
- **NEW** `/bootstrap-readme` skill — non-empty project + missing/non-compliant README path: explains the gap, runs `scripts/regenerate-readme.sh` to auto-derive what can be auto-derived, leaves `TODO:` placeholders for agent-curated sections, then surfaces the new state.
- **NEW** `/status` skill — non-empty project + compliant README path: runs `openspec list`, summarizes the last 5 commits and the last build's commit (if different), shows the last 5 `readme.traces` entries, and proposes a single highest-priority next action.
- **NEW README.md discipline** — every Tillandsias-managed project SHALL contain a `README.md` with a fixed two-section structure: `# FOR HUMANS` (header warning the file is auto-regenerated, generation timestamp with timezone, ASCII-art project-name banner, install one-liner, whimsical one-paragraph description) and `# FOR ROBOTS` (Tech Stack, Build/Runtime Dependencies, Security, Architecture, Privacy, Recent Changes, `requires_cheatsheets:` YAML block). Both sections start with the H1 header verbatim so the structural validator can find them.
- **NEW** `<project>/.tillandsias/readme.traces` — JSONL append-only ledger of agent observations during regen (`{ts, agent, observation, severity}`). Latest 50 lines are fed back as context to the next regen, so README quality monotonically improves. Committed to git for cross-machine benefit.
- **NEW per-language summarizer registry** — six initial summarizer scripts under `images/default/summarizers/` (`summarize-cargo.sh`, `summarize-nix.sh`, `summarize-package-json.sh`, `summarize-pubspec.sh`, `summarize-go-mod.sh`, `summarize-pyproject.sh`). Each reads a manifest path on argv, writes markdown to stdout, exits 0 if the manifest is present, exits >0 if not. Project-specific summarizers may live at `<project>/.tillandsias/summarizers/<name>.sh`; the dispatcher picks both up.
- **NEW** `scripts/regenerate-readme.sh` (shipped in the forge image at `/usr/local/bin/`) — the dispatcher: runs every applicable summarizer, concatenates output under `## Tech Stack` + `## Build/Runtime Dependencies`, preserves agent-curated sections, refreshes the timestamp, and writes the result.
- **NEW** `scripts/check-readme-discipline.sh` (shipped at `/usr/local/bin/`) — the structural validator: confirms required H1/H2 headers exist, the auto-regen warning is present, the timestamp parses, the `requires_cheatsheets:` YAML block is well-formed. Exits 0/non-zero so other tooling can gate on it. Output is line-by-line so a human or agent can act on each issue.
- **NEW** pre-push git hook (installed by `/startup` if missing) at `<project>/.git/hooks/pre-push` — best-effort calls `regenerate-readme.sh && git add README.md && git commit --no-edit -m 'chore(readme): regenerate at <ts>'` before push. If regen fails, falls back to a timestamp-only commit. Idempotent re-installation.
- **NEW** `cheatsheets/welcome/sample-prompts.md` — a `tier: bundled` cheatsheet listing the three (or more) curated sample prompts plus rationale. Editable without forge rebuild because it follows the cheatsheet refresh path; bumping prompts is one cheatsheet edit, not a code change.
- **NEW** `cheatsheets/welcome/readme-discipline.md` — a `tier: bundled` cheatsheet describing the README structure, FOR HUMANS / FOR ROBOTS contract, summarizer interface, and `readme.traces` schema. The skill files cite this cheatsheet via `@cheatsheet welcome/readme-discipline.md`.
- **NEW** OpenCode entrypoint shim — a small extension to `images/default/entrypoint-forge-opencode.sh` that, before `exec`ing OpenCode, writes a synthetic first user message ("run /startup") so the routing decision is taken inside the OpenCode session (visible in the chain of thought, replayable from the OpenCode session log). Survives OpenCode upgrades.
- **README requires_cheatsheets consumer** — `/startup` parses the YAML block under FOR ROBOTS, looks up each declared cheatsheet via the `cheatsheets-license-tiered` tier classifier, materializes pull-on-demand stubs for missing-but-allowlisted cheatsheets, and emits a `WARN` for missing-and-off-allowlist references. Zero new infrastructure — README becomes a new consumer of the existing tiering surface.
- **Telemetry hooks** — every routing decision and every regen invocation emits one `cheatsheet-telemetry` event with `cheatsheet = "welcome/readme-discipline.md"`, `event = "startup-routing"` (or `"readme-regen"`), `resolved_via` ∈ `{empty, bootstrap-readme, status}`, and `spec = "project-bootstrap-readme"`.

## Capabilities

### New Capabilities

- `project-bootstrap-readme`: README.md structural contract (FOR HUMANS / FOR ROBOTS), the four skills (`/startup`, `/bootstrap-readme-and-project`, `/bootstrap-readme`, `/status`), the OpenCode entrypoint shim that routes the agent into `/startup` on first prompt, the `readme.traces` JSONL ledger schema, the validator script contract, the pre-push hook contract, and the `requires_cheatsheets` consumption rule.
- `project-summarizers`: per-language manifest summarizer interface (argv contract, exit-code semantics, stdout markdown shape), the dispatcher (`scripts/regenerate-readme.sh`), the six initial summarizers, and the project-local summarizer extension path. Kept separate so the registry can grow without re-opening the bootstrap spec for every new language.

### Modified Capabilities

- `default-image`: forge image bakes the four skills under `~/.config-overlay/opencode/agent/`, the dispatcher and validator under `/usr/local/bin/`, the six summarizers under `/opt/summarizers/`. The OpenCode entrypoint shim adds one block to `entrypoint-forge-opencode.sh`. New cheatsheet directory `cheatsheets/welcome/` becomes part of the COPY surface.
- `agent-cheatsheets`: a new `welcome/` category subdirectory joins the existing seven (`runtime/`, `languages/`, `utils/`, `build/`, `web/`, `test/`, `agents/`). `INDEX.md` regeneration picks it up automatically; no schema change.

## Impact

- **Per-project filesystem additions**: `<project>/README.md` (auto-regenerated on every push), `<project>/.tillandsias/readme.traces` (JSONL, append-only, committed), `<project>/.git/hooks/pre-push` (installed by `/startup` if missing, idempotent), optional `<project>/.tillandsias/summarizers/*.sh` (project-specific summarizers, opt-in).
- **Forge image additions**: four skill markdown files under `images/default/config-overlay/opencode/agent/{startup,bootstrap-readme-and-project,bootstrap-readme,status}.md`; six summarizer scripts under `images/default/summarizers/`; two cheatsheets under `cheatsheets/welcome/{sample-prompts,readme-discipline}.md`; one dispatcher (`scripts/regenerate-readme.sh`) and one validator (`scripts/check-readme-discipline.sh`) staged into `/usr/local/bin/` at image build time. Estimated image-size impact: < 200 KB total (all bash + markdown).
- **Forge entrypoint**: `images/default/entrypoint-forge-opencode.sh` gains a small block that writes the synthetic-first-prompt shim to a known path before `exec`ing OpenCode. Other entrypoints (`entrypoint-forge-claude.sh`, `entrypoint-terminal.sh`) are unaffected — Claude and the terminal pre-date this discipline and are out of scope for v1.
- **Commit cadence**: one extra commit per `git push` per project (`chore(readme): regenerate at <ts>`). Mitigated by `--no-edit`, minimal subject, and the convention that consecutive auto-regen commits squash naturally during code review.
- **Cross-cutting consumers**: this change introduces zero new infrastructure on the proxy / git-service / inference containers. README `requires_cheatsheets:` is a NEW consumer of the existing `cheatsheets-license-tiered` tier classifier, telemetry uses the existing `cheatsheet-telemetry` EXTERNAL-tier producer, and the per-project `.tillandsias/` mount is the same one already in `forge-cache-dual`.
- **Tombstones**: none in this change. There is no superseded code; this is a wholly additive capability layer.
- **Specs touched at archive time**: NEW `project-bootstrap-readme` and `project-summarizers`; MODIFIED `default-image` (new files baked, entrypoint shim added) and `agent-cheatsheets` (new `welcome/` category subdirectory).

## Sources of Truth

- `cheatsheets/welcome/readme-discipline.md` — the README structural contract (FOR HUMANS / FOR ROBOTS) and `readme.traces` schema this proposal codifies. Created by this change; cited here for forward-compatibility with the spec's `## Sources of Truth` requirement.
- `cheatsheets/welcome/sample-prompts.md` — empty-project sample prompts. Created by this change.
- `cheatsheets/runtime/forge-paths-ephemeral-vs-persistent.md` — confirms `<project>/.tillandsias/` is per-project bind-mounted (where `readme.traces` lives) and that the project workspace is the right home for `README.md` and `.git/hooks/`.
- `cheatsheets/agents/opencode.md` — OpenCode skill / command file convention (`~/.config/opencode/agent/<name>.md` with `---\ndescription: ...\n---` frontmatter) that the four new skills follow.
- `cheatsheets/utils/jq.md` — `readme.traces` is JSONL, queryable via `jq -c`.
