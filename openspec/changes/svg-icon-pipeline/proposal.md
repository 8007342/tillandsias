## Why

The 32 SVG assets at `assets/icons/<genus>/<state>.svg` are currently dead design documentation. The system tray displays a single static PNG (`src-tauri/icons/tray-icon.png`) generated by `scripts/generate-icons.py` as a procedural rosette ‚Äî it does not use the per-genus, per-lifecycle SVGs at all. This means the visual identity the SVGs represent (8 distinct genera, 4 lifecycle states each) is never surfaced at runtime. The tray icon does not change to reflect what genus is active or what lifecycle state a container is in. The pipeline to make the SVGs load-bearing does not exist.

## What Changes

- **Build-time SVG rendering** ‚Äî `build.rs` gains `resvg` + `tiny-skia` as build dependencies and renders all 32 `assets/icons/<genus>/<state>.svg` files into PNGs at compile time; output goes to `OUT_DIR` (never checked in)
- **Tray icon PNG set** ‚Äî 3 tray icon variants (32x32) derived from the Ionantha genus: `bud.svg` for base/idle, `bloom.svg` for building/active, `dried.svg` for decay/stopped; replace the static `tray-icon.png` embed in `main.rs`
- **Window icon PNG set** ‚Äî per-genus, per-lifecycle PNGs at 32x32 and 48x48 generated for future Tauri terminal window use
- **`icon-pipeline` module** ‚Äî new Rust module in `tillandsias-core` providing `icon_png(genus, lifecycle, size)` and `tray_icon_png(state)` returning `&'static [u8]`; the existing `icons::icon_svg` function remains and is unchanged
- **`TrayIconState` enum** ‚Äî new enum in `tillandsias-core::genus` with three variants: `Base`, `Building`, `Decay`; maps to Ionantha bud/bloom/dried respectively
- **Python script preserved** ‚Äî `scripts/generate-icons.py` remains as a secondary development tool; it is no longer the sole source of tray icon PNGs

## Capabilities

### New Capabilities
- `icon-pipeline`: Build-time SVG‚ÜíPNG rendering pipeline ‚Äî `resvg`/`tiny-skia` in `build.rs`, generated PNGs in `OUT_DIR`, `include_bytes!` lookup map via `icon_png` and `tray_icon_png` functions

### Modified Capabilities
- `tray-app`: Tray icon now sourced from the SVG pipeline (Ionantha/bud by default) instead of the static procedural PNG

## Impact

- **New files**: `openspec/changes/svg-icon-pipeline/` (this change), `crates/tillandsias-core/src/icons.rs` (new module)
- **Modified files**: `src-tauri/build.rs` (add resvg rendering), `src-tauri/Cargo.toml` (add `resvg`, `tiny-skia` build-deps), `src-tauri/src/main.rs` (replace static `include_bytes!` with `icon_pipeline::tray_icon_png`), `crates/tillandsias-core/src/genus.rs` (add `TrayIconState`), `crates/tillandsias-core/src/lib.rs` (expose `icons` module)
- **Unchanged files**: `assets/icons/` (SVGs are inputs only), `scripts/generate-icons.py` (kept as dev tool), `crates/tillandsias-core/src/genus.rs::icons::icon_svg` (SVG lookup remains)
- **Not checked in**: `OUT_DIR/icons/` (generated PNGs are build artifacts)
